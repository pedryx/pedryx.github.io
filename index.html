World Simulator

C#
MonoGame
entity-component-system
procedural-generation

This project is a part of my bachelor's thesis, and it is currently a work in progress. The primary objective of this project is to explore options of the entity-component-system (ECS) pattern and to implement a game using various ECS libraries and conduct performance evaluation.


Central to this project is the abstract ECS layer, which serves as an interface for ECS libraries. The game is designed to operate with this abstract layer, ensuring its independence from any specific ECS library.


One of the core concepts of the game revolves around the simulation of a procedurally generated world. Through the utilization of GPU shaders, the entire world is procedurally generated at runtime. Within this virtual world, you'll find villages inhabited by villagers, each of whom possesses a simple AI system which utilizes behavior trees.

Run To The Stairs

C#
WPF
SkiaSharp

Unfinished reimplementation of a game called "Run to the Stairs." The entire project is divided into three parts:

  1. GameLib: This is a library designed for creating games using Monogame as a foundation. It's built to be independent of both the rendering library and the operating system.
  2. RunToTheStairs: This library actually implements the game itself by utilizing the GameLib library. Similar to GameLib, it remains detached from the rendering library and operating system.
  3. RunToTheStairs.WPF: It serves as a concrete implementation of the game for Windows. It makes use of WPF and SkiaSharp for rendering.

One of the features of this project is its abstract design, which enables the potential for expansion onto different operating systems and rendering libraries. Additionally, the project operates under the notion that a texture is defined just as something with width and height. This flexibility even opens up the possibility of implementing it within a console environment.
